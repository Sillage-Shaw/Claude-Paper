# Asynchronous Inference

Asynchronous Inference（异步推理）是一种将机器人策略推理（感知 + 动作预测）与动作执行解耦的系统架构，通过并行化计算和执行来隐藏推理延迟，提高控制响应速度。

## 基本定义

在异步推理架构中，RobotClient 负责消费动作队列中的动作并执行，PolicyServer 负责接收观测并推理生成新动作块。核心机制：

1. **队列阈值 g**：当剩余动作数 |A_t|/n < g 时，触发新的非阻塞推理请求
2. **队列聚合**：新到达的动作块与当前队列在重叠时间步上聚合
3. **相似度过滤器**：在关节空间比较观测，丢弃近重复观测以避免冗余推理

g 取值决定了异步推理的行为谱系：
- g=0：完全顺序（耗尽整个 block 后才推理）→ 有空闲期
- g=0.7（SmolVLA 默认）：消耗约 30% 旧 block 后触发新推理 → 平衡响应性和计算
- g=1：每个时间步都推理 → 响应性最强但计算代价最高

## 为什么重要

异步推理是模型无关的（model-agnostic），任何输出 action chunk 的策略都可以集成。在 SmolVLA 实验中，异步推理使任务完成时间减少约 30%（9.7s vs 13.75s），固定时间内完成任务数翻倍（19 vs 9），同时保持接近的成功率。这对于资源受限的边缘部署场景尤其有价值。
